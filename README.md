> **Computer Organization and Architecture (CSBB251) — Lab Repository**

This repository contains all laboratory experiments for the **Computer Organization and Architecture** course, implemented and documented as part of academic coursework. The experiments build a strong foundation in digital logic design, binary arithmetic, sequential circuits, and low-level debugging techniques.

---

## 📂 Repository Structure

The experiments are organized systematically for clarity and ease of navigation:

- **Experiment 01** — Half Adder, Full Adder & 4-bit Adder
  Design and verification of Half Adder, Full Adder, and 4-bit Adder circuits using Logisim.

- **Experiment 02** — 8×1 MUX & 8×3 Encoder
  Design and verification of an 8×1 Multiplexer and 8×3 Encoder using basic logic gates on Logisim.

- **Experiment 03** — 4-bit Ripple Carry Adder & Propagation Delay
  Design and simulation of a 4-bit Ripple Carry Adder and study of carry propagation delay on Logisim.

- **Experiment 04** — GNU Debugger (GDB) & Program Flow Analysis
  Debugging and tracing the execution flow of C programs using GDB commands, breakpoints, and backtrace.

- **Experiment 05** — Addressing Modes using GDB
  Study of different CPU addressing modes by disassembling and analyzing C programs with the GNU Debugger.

- **Experiment 06** — 4-bit Common Bus System
  Design and implementation of a 4-bit Common Bus System using Multiplexers and Registers on Logisim.
  
---

## 🛠️ Tools & Technologies

- **Logisim Evolution** — For digital circuit design and simulation
- **GCC (GNU Compiler Collection)** — For compiling C programs with debug symbols
- **GDB (GNU Debugger)** — For runtime debugging and execution flow analysis
- **Linux Terminal / Shell** — For GDB sessions and compilation

---

## 🎯 Objectives of the Repository

- To understand and implement fundamental combinational circuits (adders, MUX, encoders)
- To analyze carry propagation and timing delay in multi-bit arithmetic circuits
- To design and simulate a shared bus architecture using multiplexers and registers
- To gain hands-on experience with low-level C debugging using GDB
- To study CPU addressing modes at the assembly level
- To develop a structured, reproducible approach to lab documentation

---

## ✨ Key Highlights

- Well-structured and modular experiment documentation
- Practical implementation of theoretical COA concepts
- Step-by-step observations and verified truth tables
- Real GDB session logs with commands and actual terminal output
- Integration of both hardware simulation (Logisim) and software debugging (GDB)
